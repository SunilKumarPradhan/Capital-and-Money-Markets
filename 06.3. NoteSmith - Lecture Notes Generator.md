## 💰 Unrealized P&L and Mark Price

### 🔰 What is Mark Price?

> **Mark Price** is the current fair market value of a financial position, used to calculate unrealized profit and loss. It represents what you could theoretically sell your position for right now.

#### Why Mark Price Exists
- **Real-time Valuation**: Provides instant position value updates
- **Risk Management**: Helps traders monitor exposure continuously
- **Margin Calculations**: Exchanges use it to determine if you have sufficient collateral
- **Fair Pricing**: Prevents manipulation by using sophisticated pricing models

### 📊 Mark Price Calculation Methods

#### For Simple Instruments (Stocks, ETFs)
```python
# Simple mark price calculation
mark_price = current_market_price

# Example
shares_owned = 100
current_price = 150.50
position_value = shares_owned * current_price
print(f"Position Mark Value: ${position_value:,.2f}")
# Output: Position Mark Value: $15,050.00
```

#### For Complex Instruments (Futures & Options)
Mark price considers multiple factors:

| Factor | Impact | Why It Matters |
|--------|--------|----------------|
| **Underlying Asset Price** | Primary driver | Base value of the contract |
| **Time to Expiry** | Decreases over time | Time decay (theta) |
| **Market Volatility** | Increases option value | Uncertainty premium |
| **Interest Rates** | Affects carry costs | Cost of capital |
| **Dividends** | Reduces call value | Expected cash flows |

```python
# Simplified futures mark price calculation
def calculate_futures_mark_price(spot_price, risk_free_rate, time_to_expiry, dividends=0):
    """
    Calculate theoretical futures mark price
    
    Parameters:
    - spot_price: Current price of underlying asset
    - risk_free_rate: Annual risk-free interest rate (decimal)
    - time_to_expiry: Time until contract expiry (years)
    - dividends: Expected dividend payments
    
    Returns:
    - Theoretical futures price
    """
    import math
    
    # Futures price = Spot * e^(r*t) - Dividends
    futures_price = spot_price * math.exp(risk_free_rate * time_to_expiry) - dividends
    
    return futures_price

# Example
spot = 4500  # S&P 500 index
rate = 0.05  # 5% annual rate
time = 0.25  # 3 months to expiry
divs = 15    # Expected dividends

mark_price = calculate_futures_mark_price(spot, rate, time, divs)
print(f"Futures Mark Price: ${mark_price:.2f}")
# Output: Futures Mark Price: $4541.41
```

### 📈 Understanding Unrealized P&L

> **Unrealized P&L** (Profit & Loss) is the theoretical gain or loss on an open position based on current market prices. It becomes "realized" only when you close the position.

#### The Formula
```
Unrealized P&L = Market Value - Cost Basis
Market Value = Position Size × Mark Price
Cost Basis = Position Size × Entry Price
```

#### Complete P&L Calculation Example
```python
class Position:
    """Track trading position and calculate P&L"""
    
    def __init__(self, symbol, quantity, entry_price):
        self.symbol = symbol
        self.quantity = quantity
        self.entry_price = entry_price
        self.cost_basis = quantity * entry_price
    
    def calculate_unrealized_pnl(self, mark_price):
        """
        Calculate unrealized profit/loss
        
        Parameters:
        - mark_price: Current market price
        
        Returns:
        - Dictionary with P&L details
        """
        market_value = self.quantity * mark_price
        unrealized_pnl = market_value - self.cost_basis
        pnl_percentage = (unrealized_pnl / self.cost_basis) * 100
        
        return {
            'symbol': self.symbol,
            'quantity': self.quantity,
            'entry_price': self.entry_price,
            'mark_price': mark_price,
            'cost_basis': self.cost_basis,
            'market_value': market_value,
            'unrealized_pnl': unrealized_pnl,
            'pnl_percentage': pnl_percentage,
            'status': 'PROFIT' if unrealized_pnl > 0 else 'LOSS'
        }
    
    def display_pnl(self, mark_price):
        """Display formatted P&L report"""
        pnl = self.calculate_unrealized_pnl(mark_price)
        
        print(f"\n{'='*50}")
        print(f"Position Report: {pnl['symbol']}")
        print(f"{'='*50}")
        print(f"Quantity: {pnl['quantity']:,} shares")
        print(f"Entry Price: ${pnl['entry_price']:.2f}")
        print(f"Current Mark Price: ${pnl['mark_price']:.2f}")
        print(f"-" * 50)
        print(f"Cost Basis: ${pnl['cost_basis']:,.2f}")
        print(f"Market Value: ${pnl['market_value']:,.2f}")
        print(f"-" * 50)
        print(f"Unrealized P&L: ${pnl['unrealized_pnl']:,.2f} ({pnl['pnl_percentage']:.2f}%)")
        print(f"Status: {pnl['status']}")
        print(f"{'='*50}\n")

# Example Usage
# Scenario 1: Profitable position
long_position = Position(symbol="AAPL", quantity=500, entry_price=150.00)
long_position.display_pnl(mark_price=165.50)

# Scenario 2: Losing position

short_position.display_pnl(mark_price=275.00)
```

### 🎓 Key Concepts to Remember

**Why "Unrealized"?**
- The P&L is **theoretical** until you close the position
- Prices can change before you exit
- You cannot spend unrealized profits
- Unrealized losses don't require cash payment (yet)

**When It Becomes Realized:**
```python
def close_position(position, exit_price):
    """
    Close position and realize P&L
    
    Parameters:
    - position: Position object
    - exit_price: Price at which position is closed
    
    Returns:
    - Realized P&L
    """
    realized_pnl = (exit_price - position.entry_price) * position.quantity
    
    print(f"Position CLOSED at ${exit_price:.2f}")
    print(f"Realized P&L: ${realized_pnl:,.2f}")
    print(f"This P&L is now FINAL and affects your cash balance")
    
    return realized_pnl

# Example
position = Position("NVDA", 100, 450.00)
close_position(position, 475.00)
# Output: Realized P&L: $2,500.00
```

3. **Risk Limits**: Firms set maximum unrealized loss thresholds
4. **Performance Reporting**: Daily P&L reports for traders and investors

## 🔄 Order Matching Systems

### 🔰 What is Order Matching?

> **Order Matching** is the automated process of pairing buy and sell orders for the same security when they meet specific compatibility criteria. It's the core mechanism that enables trading on modern exchanges.

#### The Dance Analogy
Think of order matching as a sophisticated dating app for trades:
- **Buy orders** are looking for sell orders
- **Sell orders** are looking for buy orders
- The "matching algorithm" acts as the matchmaker
- Compatible orders get "paired" and execute

### 📋 Order Compatibility Rules

#### Basic Compatibility Condition
```
A buy order and sell order are compatible when:
Buyer's Maximum Price ≥ Seller's Minimum Price
```

```python
class Order:
    """Represents a trading order"""
    
    def __init__(self, order_id, side, symbol, quantity, price, order_type="LIMIT"):
        self.order_id = order_id
        self.side = side.upper()  # "BUY" or "SELL"
        self.symbol = symbol
        self.quantity = quantity
        self.price = price
        self.order_type = order_type
        self.timestamp = None  # Set when order enters book
    
    def __repr__(self):
        return f"{self.side} {self.quantity} {self.symbol} @ ${self.price}"

def are_orders_compatible(buy_order, sell_order):
    """
    Check if buy and sell orders can be matched
    
    Parameters:
    - buy_order: Order object with side="BUY"
    - sell_order: Order object with side="SELL"
    
    Returns:
    - Boolean indicating compatibility
    """
    # Must be same symbol
    if buy_order.symbol != sell_order.symbol:
        return False
    
    # Price compatibility: buyer willing to pay >= seller asking
    if buy_order.price >= sell_order.price:
        return True
    
    return False

# Example
buy = Order("B001", "BUY", "AAPL", 100, 150.50)
sell = Order("S001", "SELL", "AAPL", 100, 150.00)

if are_orders_compatible(buy, sell):
    print(f"✓ MATCH: {buy} can match with {sell}")
    print(f"Execution Price: ${sell.price} (seller's price)")
else:
    print("✗ NO MATCH: Orders not compatible")

# Output: ✓ MATCH: BUY 100 AAPL @ $150.5 can match with SELL 100 AAPL @ $150.0
```

### ⚙️ Matching Algorithms

Modern exchanges use sophisticated algorithms to prioritize and match orders fairly.

#### 1. Price-Time Priority (FIFO - First In, First Out)

> **Most Common Algorithm**: Orders are matched first by best price, then by time of arrival.

**Priority Rules:**
1. **Best Price First**: Highest buy price, lowest sell price
2. **Time Priority**: Earlier orders get priority at same price level

```python
from datetime import datetime
from collections import defaultdict
import heapq

class OrderBook:
    """Simple order book with price-time priority matching"""
    
    def __init__(self, symbol):
        self.symbol = symbol
        self.buy_orders = []   # Max heap (highest price first)
        self.sell_orders = []  # Min heap (lowest price first)
        self.order_counter = 0
        
    def add_order(self, order):
        """
        Add order to book and attempt matching
        
        Parameters:
        - order: Order object
        """
        order.timestamp = datetime.now()
        self.order_counter += 1
        
        print(f"\n[{order.timestamp.strftime('%H:%M:%S.%f')[:-3]}] New Order: {order}")
        
        if order.side == "BUY":
            # Try to match with existing sell orders
            self._match_buy_order(order)
        else:
            # Try to match with existing buy orders
            self._match_sell_order(order)
    
    def _match_buy_order(self, buy_order):
        """Match incoming buy order with sell orders"""
        remaining_qty = buy_order.quantity
        
        while remaining_qty > 0 and self.sell_orders:
            # Peek at best sell order (lowest price)
            best_sell = self.sell_orders[0]
            
            # Check compatibility
            if buy_order.price >= best_sell.price:
                # Execute trade
                sell_order = heapq.heappop(self.sell_orders)
                trade_qty = min(remaining_qty, sell_order.quantity)
                trade_price = sell_order.price  # Seller's price (resting order)
                
                print(f"  ✓ TRADE EXECUTED: {trade_qty} @ ${trade_price}")
                print(f"    Buyer: {buy_order.order_id} | Seller: {sell_order.order_id}")
                
                remaining_qty -= trade_qty
                sell_order.quantity -= trade_qty
                
                # If sell order partially filled, put it back
                if sell_order.quantity > 0:
                    heapq.heappush(self.sell_orders, sell_order)
            else:
                # No more matches possible
                break
        
        # Add remaining buy order to book
        if remaining_qty > 0:
            buy_order.quantity = remaining_qty
            # Use negative price for max heap behavior

                          (-buy_order.price, buy_order.timestamp, buy_order))
